# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| Latest  | :white_check_mark: |
| < Latest | :x:               |

Only the latest published version of `@felloh-org/payment-sdk` receives security updates.

## Reporting a Vulnerability

If you discover a security vulnerability in this SDK, please report it responsibly. **Do not open a public GitHub issue.**

Email **developers@felloh.com** with:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fix (optional)

You should receive an acknowledgement within **48 hours**. We will work with you to understand the issue and coordinate a fix and disclosure timeline.

## Disclosure Policy

- We aim to release a patch within **7 days** of confirming a vulnerability.
- We will credit reporters in the release notes unless they prefer to remain anonymous.
- We ask that you do not publicly disclose the issue until a fix has been released.

## Scope

This policy covers the `@felloh-org/payment-sdk` npm package and its source code in this repository. For vulnerabilities in the Felloh payment service itself (the hosted iframe), please contact **security@felloh.com** directly.
